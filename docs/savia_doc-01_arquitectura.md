### [CUERPO DEL DOCUMENTO]
#### Resumen del Concepto
El estandar Open Badges v3 (OBv3) representa la evolucion definitiva de
las credenciales digitales, alineandose con la especificacion oficial de
Credenciales Verificables (VC) del W3C. Este marco tecnologico permite a
la universidad emitir titulos inmutables, portables y verificables de
forma criptografica, eliminando la dependencia de bases centralizadas.

#### Justificacion Tecnica y Evidencia
La transicion hacia OBv3 y W3C VC es fundamental para el proyecto SAVIA.
El Digital Credentials Consortium (DCC) provee la arquitectura base para
este ecosistema. Segun la documentacion oficial, el motor asegura que
cada credencial este firmada digitalmente y estructurada bajo JSON-LD.
URL: github.com/digitalcredentials/docs
Cita: "The DCC stack provides a complete solution for issuing and
verifying W3C Verifiable Credentials."

#### Configuracion / Logica Acordada
1. Desplegar el motor DCC como nucleo del nodo emisor de la UNR.
2. Configurar el DID (did:web:id.unr.edu.ar) como el identificador
soberano de la institucion en la red descentralizada.
3. Estructurar las plantillas de titulos (Credential Templates)
cumpliendo estrictamente con el esquema Open Badges v3.
